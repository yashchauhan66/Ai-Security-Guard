# Support Process

## Support Contact
- Primary support email for pilot customers: `support@soter-example.com`
- Security issue reporting: `security@soter-example.com`

## Bug Report Flow
1. User reports an issue (e.g., false positive block) via the extension popup or to their internal IT helpdesk.
2. The internal IT helpdesk can inspect the event in the Soter Admin Control Plane.
3. If it's a bug in the extension, the IT team escalates to `support@soter-example.com` with the event ID.

## Pilot Customer Onboarding
- Dedicated onboarding sessions will be held for the initial beta pilot cohort.
- Admins will receive secure enrollment tokens to distribute to their fleet via MDM.

## Rollback Process
- If an enterprise experiences critical blockage, admins can use the "Emergency Lockdown" toggle or disable the extension globally via MDM.

## Incident Escalation
- Critical P0 incidents (e.g., extension crashing the browser) have a 2-hour response SLA for paid pilot customers.
